# GlassGPT 5.3.0 Industry-Leading Hardening Release Plan

## Summary

- `5.3.0` is a quality-first major improvement release whose purpose is to lift the codebase to a genuinely top-tier level, then prove it with a second full multi-dimensional rescore using the same 20-category rubric.
- This plan explicitly absorbs every issue from the current review, your 20 scored categories, your top-10 priorities, and the additional verified findings from the repo audit.
- `5.3.0` may include versioned breaking backend/app contract changes where needed, but the release must still be operationally safe: staged backend rollout, scripted promotion, scripted TestFlight upload, and a written compatibility/migration path.
- Release exit bar:
  - Overall rescore must be `>= 17.5/20`.
  - No category may finish below `16/20`.
  - Architecture, Security, Testing, CI/CD, Error Handling, Maintainability, and API Design must each reach `>= 18/20`.
  - The final scorecard must be written to `/Applications/GlassGPT/docs/audit-5.3.0.md` and linked from the live tracker before release.
  - An independent `gpt-5.4` `xhigh` rubric review must also pass; if it fails, every blocker from that review becomes mandatory implementation work before release can resume.

## Critical Path Priority Order

- `P0-001` Make backend conversation configuration authoritative and end-to-end real: model, reasoning effort, agent worker effort, and service tier must be stored on the backend, round-tripped through contracts, surfaced in sync APIs, and actually used in run execution.
- `P0-002` Remove Chat/Agent duplication from controllers, views, stream handlers, and payload decoding by moving to shared composition and shared run-surface support types.
- `P0-003` Replace ambiguous CORS string matching with parsed-origin validation backed by an environment allowlist and explicit staging/dev origins.
- `P1-001` Replace the current in-memory rate limiter with a persistent Cloudflare-native limiter and actually mount it in the HTTP app for authenticated and unauthenticated routes.
- `P1-002` Decompose `chat-run-service.ts` so chat execution matches the agent side’s level of separation.
- `P1-003` Migrate device identity storage from `UserDefaults` to Keychain with one-time migration.
- `P1-004` Make SSE resumability real end-to-end: frame IDs, stored resume cursor, client replay header, server replay path, and integration tests.
- `P1-005` Wire the iOS retry layer into real request call sites and add deterministic retry tests.
- `P2-001` Fix CI/runtime version drift for Node and document the supported toolchain truthfully.
- `P2-002` Add SSE end-to-end integration tests and concurrency stress tests.
- `P2-003` Add pagination to conversation listing and move filtering/windowing out of full in-memory loads.
- `P2-004` Add staged backend deployment and scripted rollback instead of direct production-only deploys.
- `P3-001` Remove `AnyView` from the root conversation surfaces and add visual snapshot regression tests.

## Required Public Interface And Contract Changes

- `ConversationDTO` and `ConversationDetailDTO` become the canonical source of conversation configuration.
- `CreateConversationRequestDTO` must carry full configuration for the selected mode.
- Add `PATCH /v1/conversations/:conversationId/configuration` so changing selectors updates the authoritative backend record before future runs.
- `StartAgentRunRequestDTO` remains prompt-focused, but agent runs must consume the stored conversation configuration unless an explicit per-run override is intentionally added.
- `CreateMessageRequestDTO` may remain content-only once conversation configuration is authoritative on the server.
- `GET /v1/conversations` becomes a paginated response envelope with `items`, `nextCursor`, and `hasMore`.
- SSE responses must emit stable `id:` fields and support resume from the last delivered event ID.
- Health/connection-check responses must expose backend release/contract compatibility so the iOS app can reject incompatible servers cleanly.
- The release automation contract becomes script-only:
  - `/Applications/GlassGPT/scripts/deploy_backend.sh` must support staged deploy, migration backup, smoke checks, and rollback behavior.
  - `/Applications/GlassGPT/scripts/release_testflight.sh` must remain the TestFlight publish path, but be modernized and made `5.3.0`-aware.
  - Add a new orchestrator script, `/Applications/GlassGPT/scripts/release_5_3.sh`, as the only approved release entrypoint; it must call backend deploy/promotion and TestFlight publish in order.

## Implementation Workstreams

- **Workstream 1: End-to-End Product Truth**
  - Make backend conversation config authoritative in D1/domain/contracts.
  - Sync existing local SwiftData config to backend on first `5.3.0` launch when a matching `serverID` exists and the backend record is still unset/default.
  - Remove all “UI-only” configuration behavior so selector changes affect the next real run on every device.
  - Add compatibility/version checks so a mismatched backend is surfaced as a first-class error state, not silent drift.
- **Workstream 2: Architecture And Deduplication**
  - Remove `NativeChatBackendCore -> NativeChatUI` by relocating shared non-UI symbols into a non-UI layer that already represents presentation or logic.
  - Keep `BackendChatController` and `BackendAgentController` only as thin `@Observable` surface stores with minimal public mutable state.
  - Move networking orchestration, persistence coordination, stream lifecycle, and event projection into dedicated collaborators.
  - Replace mode-pair duplication with shared scaffolds for conversation shell, composer, stream state, payload types, and serializer helpers.
- **Workstream 3: Backend Hardening**
  - Split chat execution into smaller workflow/application units matching the agent path’s decomposition style.
  - Remove duplicated service interface declarations from HTTP composition and reuse application-layer types directly.
  - Replace unsafe `JSON.parse(...) as ...` mapper paths with runtime-validated decoding.
  - Add conversation pagination and route-level performance limits.
- **Workstream 4: Security And Identity**
  - Replace suffix-based CORS checks with parsed host validation from an allowlist.
  - Replace in-memory rate limiting with a persistent Cloudflare-native design keyed by user and anonymous client identity.
  - Move device identity to Keychain and migrate legacy values.
  - Add D1 backup/export before remote migrations and a documented restore path.
- **Workstream 5: Streaming Reliability And Performance**
  - Make SSE resume real.
  - Differentiate connection-start failures from mid-stream failures in `BackendSSEStream`.
  - Replace global/shared circuit breaker behavior with a keyed sliding-window design to avoid one user poisoning others.
  - Fix parser/cache heuristics that cause unnecessary full reparses on common LLM patterns.
  - Add memory-pressure-aware cache trimming for file/markdown caches.
- **Workstream 6: Test And Gate Uplift**
  - Add SSE integration coverage, visual snapshots, concurrency stress tests, rollback/deploy tests, migration failure tests, config propagation tests, and retry-path tests.
  - Upgrade smoke/coverage tests into assertion-rich behavioral tests.
  - Extract shared backend test doubles/harnesses to remove repetitive boilerplate.
  - Add TypeScript coverage reporting and a backend coverage gate.
  - Add explicit regression coverage for malformed stream snapshot handling and persistence recovery paths where the independent review found error-handling gaps.
- **Workstream 7: Docs, DX, And Scorecard Truth**
  - Rewrite architecture, testing, security, release, branch, and contributing docs so they describe the real `5.3.0` system.
  - Add a backend local-development guide and `.env.example`-style setup documentation.
  - Keep zero external Swift dependencies unless a change is explicitly approved in the scorecard and justified against ADR-007.
  - Publish the final `5.3.0` scorecard with evidence rather than narrative claims.
  - Eliminate all current-release Beta 5.0 residue from active docs, workflows, and source comments that are still part of the shipping/release path.
- **Workstream 8: Release And Operational Maturity**
  - Introduce staging/preview backend environments before production deploy.
  - Add scripted smoke-check promotion and scripted rollback.
  - Modernize TestFlight upload tooling and branch/version validation.
  - Add native observability improvements without breaking the zero-Swift-dependency principle.
  - Align release/workflow metadata, branch policy, and version strings so the release-readiness gate, docs, and current branch all describe the same `5.3.0` line.

## Category-By-Category Target Plan

1. **Modular Architecture: `16 -> 19`.** Preserve the 21 Swift-module layering, zero-dependency `ChatDomain`, ADR-007, and dependency-cruiser rules. Remove `NativeChatBackendCore -> NativeChatUI`, stop treating controllers as module hubs, and make dependency rules fail if backend core imports UI again.
2. **SOLID: `13 -> 18`.** Preserve DI factories, `@Observable`, and storage abstractions. Refactor chat/agent controllers into thin view-facing stores; hide raw mutable internals behind intent methods or `private(set)` state; replace hardcoded mode switches with strategy/projector types.
3. **Code Duplication: `11 -> 18`.** Eliminate the 500+ line Chat/Agent duplication by extracting shared action coordination, shared view scaffolding, shared stream payload models, shared event projector infrastructure, and shared encode/decode helpers for persisted payloads.
4. **Swift Modernity: `18 -> 19`.** Preserve Swift 6, strict concurrency, `@Observable`, `async/await`, and iOS 26 posture. Replace remaining `DispatchQueue.main.async` usages with `Task { @MainActor }` or actor-safe equivalents; remove `AnyView` type erasure from root surfaces where static view composition is possible.
5. **Concurrency Safety: `16 -> 18`.** Preserve `@MainActor` isolation, `weak self`, stale-selection guards, and `StreamEventBatcher` cancellation. Replace non-structured long-lived tasks with explicit parent-child orchestration where practical, differentiate SSE error classes, and route flush failures into explicit logging/error sinks instead of silent release-mode drops.
6. **UI Architecture: `15 -> 18`.** Preserve `StreamingTextView`, `MarkdownBlockCache`, and the existing UI test scaffolding. Remove `AnyView`, add snapshot coverage for primary chat/agent/settings/history states, and move `ProjectionCacheRepository` filtering/windowing into queries instead of full-load-then-filter behavior.
7. **Backend Architecture: `16 -> 18`.** Preserve the Domain → Application → Adapter → HTTP layering, readonly interfaces, Zod validation, and Cloudflare Workflows. Split chat execution into smaller units, delete duplicated interface declarations in HTTP composition, and validate parsed mapper payloads at runtime.
8. **API Design: `16 -> 18`.** Preserve REST resource clarity, status-code discipline, Zod boundaries, SSE batching, heartbeat, and initial state replay. Add conversation pagination, centralize auth token parsing, preserve enough SSE error context for actionable client behavior, and make configuration/state contracts truthful and round-trippable.
9. **Security: `15 -> 18`.** Preserve encrypted credentials, keyed refresh token storage, Keychain usage, parameterized SQL, and log sanitization. Replace the origin matcher, ship the persistent limiter, migrate device identity to Keychain, and explicitly cover anonymous abuse and limiter eviction semantics.
10. **Performance Optimization: `16 -> 18`.** Preserve `StreamingTextCache`, batching, micro-buffering, file cache, signposts, and regression baselines. Fix the `***`/unclosed-span heuristic path, replace the global circuit breaker design, and add memory-pressure-aware adaptive cache trimming.
11. **Test Coverage: `14 -> 18`.** Preserve the current breadth across persistence, networking, domain, contracts, architecture, and accessibility. Add SSE integration tests, snapshot tests, package-level tests for under-covered areas, and concurrency stress tests.
12. **Test Quality: `15 -> 18`.** Preserve URLProtocol-backed HTTP tests, in-memory SwiftData tests, contract mirror tests, and real temp-file cache tests. Replace smoke-only assertions with semantic assertions, consolidate repeated backend test doubles into reusable harnesses, and deepen retry/recovery path testing.
13. **CI/CD Pipeline: `17 -> 19`.** Preserve the four-lane structure, hard gates, infra self-tests, and maintainability checks. Fix Node-version drift, add backend coverage gating, and ensure `todo.md`/scorecard/release evidence are part of release-readiness.
14. **Documentation Quality: `15 -> 18`.** Preserve ADRs, release playbooks, and changelog discipline. Rewrite stale 4.9/5.0 docs, update `SECURITY.md`, expose current API/contract documentation paths, and ensure docs are generated or validated from the actual `5.3.0` architecture.
15. **Developer Experience: `13 -> 17`.** Preserve local full-CI capability, hooks, and practical contribution guidance. Add backend local-dev docs, an environment template, current branch guidance, and toolchain bootstrapping instructions that reduce onboarding friction without lowering the quality bar.
16. **Release Management: `17 -> 19`.** Preserve the strength of the current release wrapper, artifact validation, IPA checks, and git/tag automation. Replace deprecated upload tooling, add backend/app compatibility coordination, and make the whole production/TestFlight path script-driven and version-line aware for `5.3.0`.
17. **Dependency Management: `18 -> 19`.** Preserve zero external Swift dependencies, strict TS settings, Biome rules, and pnpm pinning. Extend Dependabot coverage to npm/pnpm, add vulnerability scanning to CI, and keep dependency updates tied to the release scorecard.
18. **Error Handling: `14 -> 18`.** Preserve unified backend `ApplicationError`, global HTTP mapping, and schema-validated error responses. Stop silently swallowing broadcast failures, surface safe SSE error context, replace `NSLog` fallback paths with project-standard logging, and add migration/corruption recovery tests.
19. **Maintainability: `15 -> 19`.** Preserve current Swift file-size discipline and automated maintainability policy. Remove Chat/Agent duplication, introduce equivalent maintainability budgets for TypeScript, normalize inconsistent naming (`isStreaming` vs `isRunning`, `syncMessages` vs `syncVisibleState`), and keep oversized backend files from regressing.
20. **Operational Maturity: `13 -> 17`.** Preserve MetricKit, signposts, and performance regression detection. Add staging/preview environments, scripted rollback, stronger backend observability/alerting, and D1 backup/restore integration around migrations.

## Assumptions And Defaults

- `5.3.0` is a major quality release, not a mixed feature release.
- Versioned breaking backend/app contract changes are allowed where needed, but must be coordinated inside the same backend+iOS release train and recorded in the scorecard.
- The zero external Swift dependency principle remains a non-regression requirement unless the scorecard explicitly approves an exception.
- The final release will be judged by the second rubric-based rescore, not by subjective claims that quality “feels better”.
- The live tracker is `/Applications/GlassGPT/todo.md`, the release plan is `/Applications/GlassGPT/5.3.0-plan.md`, the release evidence report is `/Applications/GlassGPT/docs/audit-5.3.0.md`, backend promotion is script-only, and TestFlight promotion is script-only.
