# GlassGPT 5.3.3 Feature Parity & Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restore feature parity with stable-4.12 (image upload, file upload, sandbox file download, tool call UI) on the 5.3 backend architecture, and fix all remaining streaming/polling UX issues discovered during the 5.3.2 hotfix session.

**Architecture:** The 4.12 branch was a pure client-side architecture where the iOS app called OpenAI directly. The 5.3 branch routes everything through Cloudflare Workers. This plan redesigns each feature for the backend-proxied architecture: images are base64-inlined in the backend's OpenAI request, files are uploaded to OpenAI via the backend (proxied Files API), sandbox file downloads are proxied through a new backend endpoint, and tool call state is persisted to D1 so the 250ms polling loop can display intermediate states.

**Tech Stack:** Swift 6 / iOS 26, Cloudflare Workers (Hono), D1, R2, OpenAI Responses API, Zod validation

**Reference branch:** `stable-4.12` (read-only reference for UI patterns, NOT a merge source)

## Context: Issues Fixed in 5.3.2 (Already Shipped)

These are resolved and should NOT be re-opened:

| Issue | Root Cause | Fix |
|---|---|---|
| Short messages ("你好", "hi") produce zero output | OpenAI sends `[DONE]` without `response.completed`; consumer never flushed `pendingTextDelta` | Consumer-side flush after for-await loop exit |
| No streaming (content arrives all at once) | CF DO→Worker RPC buffers SSE; `Content-Encoding: identity` and `encodeBody: 'manual'` ineffective | Replaced SSE with 250ms high-frequency polling against D1 |
| D1 read-replica staleness on finalize | `finalizeVisibleRun()` reads from stale D1 replica | Client snapshots `currentStreamingText` as fallback |
| Missing `done` SSE event in error/cancel paths | `catch` block and cancellation check never broadcast `done` | Added `done` broadcast in all terminal paths |
| No conversation context (multi-turn) | `buildChatExecutionRequest` sent only current message text | Load conversation history, pass as message array with `output_text` type for assistant messages |
| `timeoutIntervalForResource` killing SSE | Shared URLSession with 60s resource timeout | Dedicated `sseURLSession` with infinite timeout (retained for future SSE re-enablement) |
| `isThinking` and `activeToolCalls` not set during polling | Live overlay properties only populated by SSE events | `applyLiveOverlayFromPolledMessages()` syncs from polled message data |

```typescript
  input: string | readonly StreamingConversationMessage[],
  imageBase64?: string,
  fileIds?: readonly string[],
): string | ResponsesApiInputMessage[] => {
  // Simple string input with optional image/files
  if (typeof input === 'string') {
    const hasAttachments = (imageBase64 && imageBase64.length > 0) ||
                           (fileIds && fileIds.length > 0);
    if (!hasAttachments) {
      return input;
    }
    // Convert to structured format for multi-modal
    const parts: Array<Record<string, unknown>> = [
      { text: input, type: 'input_text' },
    ];
    if (imageBase64) {
      parts.push({
        image_url: `data:image/jpeg;base64,${imageBase64}`,
        type: 'input_image',
      });
    }
    for (const fileId of fileIds ?? []) {
      parts.push({ file_id: fileId, type: 'input_file' });
    }
    return [{ content: parts, role: 'user' as const }];
  }

  // Message array input — image/files only attach to the LAST user message
  return input.map((message, index) => {
    const isLastUser = message.role === 'user' &&
      index === input.length - 1;
    const contentType = message.role === 'assistant'
      ? 'output_text'
      : 'input_text';
    const parts: Array<Record<string, unknown>> = [
      { text: message.content, type: contentType },
    ];
    if (isLastUser && imageBase64) {
      parts.push({
        image_url: `data:image/jpeg;base64,${imageBase64}`,
        type: 'input_image',
      });
    }
    if (isLastUser) {
      for (const fileId of fileIds ?? []) {
        parts.push({ file_id: fileId, type: 'input_file' });
      }
    }
    return { content: parts, role: message.role };
  });
```

```typescript
import { requireAuthenticatedSession } from '../require-authenticated-session.js';
import { asBackendRuntimeContext } from '../runtime-context.js';
import type { BackendServices } from '../services.js';
import type { BackendApp } from '../types.js';

export const installFileRoutes = (app: BackendApp, services: BackendServices): void => {
  // Upload a file to OpenAI via the backend (proxies multipart form data)
  app.post('/v1/files/upload', async (context) => {
    const session = await requireAuthenticatedSession(context, services);
    const env = asBackendRuntimeContext(context.env);

    const formData = await context.req.formData();
    const file = formData.get('file') as File | null;
    const purpose = (formData.get('purpose') as string) ?? 'user_data';

    if (!file) {
      return context.json({ error: 'file_required' }, 400);
    }

    const credential = await services.credentialService.getCredential(env, session.userId, 'openai');
    if (!credential) {
      return context.json({ error: 'openai_credential_unavailable' }, 403);
    }
    const apiKey = await services.credentialService.decryptApiKey(env, credential);

    // Proxy to OpenAI Files API
    const openAIForm = new FormData();
    openAIForm.append('file', file, file.name);
    openAIForm.append('purpose', purpose);

    const openAIResponse = await fetch('https://api.openai.com/v1/files', {
      method: 'POST',
      headers: { Authorization: `Bearer ${apiKey}` },
      body: openAIForm,
    });

    if (!openAIResponse.ok) {
      const errorText = await openAIResponse.text();
      return context.json(
        { error: 'openai_file_upload_failed', detail: errorText },
        openAIResponse.status as 400 | 500,
      );
    }

    const result = (await openAIResponse.json()) as { id: string; filename: string; bytes: number };
    return context.json({ fileId: result.id, filename: result.filename, bytes: result.bytes }, 201);
  });
};
```

```swift
import Foundation

@MainActor
public extension BackendClient {
    /// Uploads a file to OpenAI through the backend proxy.
    /// Returns the OpenAI file ID on success.
    func uploadFile(
        data: Data,
        filename: String,
        mimeType: String
    ) async throws -> String {
        try await refreshSessionIfNeeded(for: .required)

        var request = URLRequest(url: environment.baseURL.appendingPathComponent("/v1/files/upload"))
        request.httpMethod = "POST"
        request.timeoutInterval = environment.timeoutInterval

        let boundary = UUID().uuidString
        request.setValue("multipart/form-data; boundary=\(boundary)", forHTTPHeaderField: "Content-Type")
        if let auth = makeAuthorizationHeader(sessionStore: sessionStore) {
            request.setValue(auth, forHTTPHeaderField: "Authorization")
        }

        var body = Data()
        body.append("--\(boundary)\r\n".data(using: .utf8)!)
        body.append("Content-Disposition: form-data; name=\"file\"; filename=\"\(filename)\"\r\n".data(using: .utf8)!)
        body.append("Content-Type: \(mimeType)\r\n\r\n".data(using: .utf8)!)
        body.append(data)
        body.append("\r\n--\(boundary)\r\n".data(using: .utf8)!)
        body.append("Content-Disposition: form-data; name=\"purpose\"\r\n\r\n".data(using: .utf8)!)
        body.append("user_data".data(using: .utf8)!)
        body.append("\r\n--\(boundary)--\r\n".data(using: .utf8)!)
        request.httpBody = body

        let (responseData, response) = try await urlSession.data(for: request)
        guard let httpResponse = response as? HTTPURLResponse,
              (200..<300).contains(httpResponse.statusCode) else {
            throw BackendAPIError.networkFailure("File upload failed")
        }

        struct UploadResponse: Decodable { let fileId: String }
        let result = try JSONDecoder().decode(UploadResponse.self, from: responseData)
        return result.fileId
    }
}
```

## Task 5: Tool Call UI — Async State Machine for Polling Architecture

**Design:** The core problem is that with 250ms polling, tool call status transitions happen between polls. The solution is a **client-side tool call state machine** that infers state from the polled data using temporal heuristics rather than relying on real-time status updates.

### State Machine Design

```
                    ┌─────────────────────────────────┐
                    │     ToolCallDisplayState         │
                    ├─────────────────────────────────┤
                    │ .hidden                         │ ← no tool calls in message
                    │ .active(type, since)            │ ← tool call appeared, show indicator
                    │ .completing(type, completedAt)  │ ← status=completed, hold indicator
                    │                                 │   for 2s grace period
                    │ .done(type)                     │ ← grace period expired, hide indicator
                    └─────────────────────────────────┘

Transitions:
  .hidden     → .active      : toolCalls array gains entry with matching type
  .active     → .completing  : all entries of this type have status=completed
  .completing → .done        : 2 seconds elapsed since completedAt
  .completing → .active      : new entry of this type appears with status!=completed
  .done       → .active      : new entry of this type appears (new tool invocation)
```

The 2-second grace period in `.completing` ensures the indicator remains visible even though polling only sees `completed` status. This matches the user's mental model: "the search happened, I saw the indicator."

```swift
/// Tracks when each tool call type was first observed, enabling a grace
/// period so indicators remain visible even when polling only sees
/// the completed state.
private var toolCallFirstSeen: [String: Date] = [:]  // keyed by tool call ID

func applyLiveOverlayFromPolledMessages() {
    guard let lastAssistant = messages.last(where: { $0.role == .assistant }) else { return }

    // Thinking state
    let hasThinking = !(lastAssistant.thinking?.isEmpty ?? true)
    let hasContent = !lastAssistant.content.isEmpty
    isThinking = hasThinking && !hasContent
    if hasThinking { currentThinkingText = lastAssistant.thinking ?? "" }
    if hasContent { currentStreamingText = lastAssistant.content }

    // Tool calls — mark newly seen tool calls with timestamp
    let now = Date()
    for toolCall in lastAssistant.toolCalls {
        if toolCallFirstSeen[toolCall.id] == nil {
            toolCallFirstSeen[toolCall.id] = now
        }
    }

    // Build display list: include completed tool calls that were
    // first seen within the last 3 seconds (grace period)
    let gracePeriod: TimeInterval = 3.0
    var displayCalls = lastAssistant.toolCalls.map { toolCall -> ToolCallInfo in
        let firstSeen = toolCallFirstSeen[toolCall.id] ?? now
        let age = now.timeIntervalSince(firstSeen)
        if toolCall.status == .completed && age < gracePeriod {
            // Override status to show as active during grace period
            return ToolCallInfo(
                id: toolCall.id,
                type: toolCall.type,
                status: .inProgress,
                code: toolCall.code,
                results: toolCall.results,
                queries: toolCall.queries
            )
        }
        return toolCall
    }
    activeToolCalls = displayCalls
```
